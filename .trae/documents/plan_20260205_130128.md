以下是详细的优化实施计划：

### 1. 交互与动画升级 (Interaction & Motion)

*   **Loading 动画修复**：
    *   优化 CSS 布局，防止猫咪动画与文字在不同屏幕尺寸下重叠。
*   **状态御守 (StatusCard) 2.0 - “祈愿与抽签”**：
    *   **折叠/展开交互**：御守平时只显示 Emoji 和名字，点击或下拉时，如卷轴般展开详细状态卡片（解决字看不清的问题）。
    *   **抽签系统 (Omikuji)**：
        *   重做“刷新”按钮为**“签筒”**。
        *   **摇签动画**：点击播放摇晃签筒掉出签子的 CSS/GSAP 动画。
        *   **签文设计**：包含“大吉/中吉/小吉” + 浪漫签文（如“宜想念”、“忌熬夜”）。
        *   **彩蛋奖品**：设置低概率（如 5-10%）触发“特别奖”，弹出“奶茶兑换券”或“愿望兑换券”卡片，可截图兑换。

### 2. 视觉样式修正 (Visual Refinement)

*   **庆典倒计时 (BirthdayCountdown)**：
    *   **样式适配**：将白色文字改为**深墨色**或**烫金**效果，适配浅色和纸背景。
    *   **容器优化**：增加一个半透明的白色底板或日式边框，提升可读性。
*   **动态时光 (Timeline) - “手帐笔记”**：
    *   **笔记本样式**：给时间线组件添加一个**“活页笔记本”**的外观容器（带螺旋线圈和纸张纹理）。
    *   **手写感**：内容区域模拟手写笔记的排版。

### 3. 功能板块优化 (Functional Modules)

*   **留言板 (InteractiveZone)**：
    *   **便签墙风格**：将留言区设计成**软木板**或**挂绳便签**样式，留言像便利贴一样贴在上面。
    *   **后端预留**：完善前端数据结构，模拟 API 请求（`mockApi`），为未来接入数据库做准备。

### 4. 移动端布局重构 (Mobile Layout)

*   **底部导航实装**：
    *   激活底部导航栏的三个图标（🏠 首页、📸 时光/相册、💌 留言）。
    *   **Tab 切换模式**：为了避免“一条滑到底”的单调感，我们将页面改为 **Tab 切换结构**。
        *   **首页**：只展示计时器、状态御守、倒计时（核心信息）。
        *   **时光**：展示照片墙和动态笔记（沉浸式浏览）。
        *   **留言**：独立的便签墙页面。
    *   这样不仅解决了页面过长的问题，也让每个板块有独立的展示空间，设计感更强。
